# neo-design-patterns-hw-02

Рефакторинг системи повідомлень відповідно до принципів SOLID.

1. SRP: кожен клас відповідає лише за одну відповідальність.
2. OCP: нові канали додаються без зміни існуючого коду.
3. LSP: усі канали реалізують спільний інтерфейс.
4. ISP: використано окремі інтерфейси для кожної відповідальності.
5. DIP: залежності передаються через інтерфейси, а не створюються напряму.
